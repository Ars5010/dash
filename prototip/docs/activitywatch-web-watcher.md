## Web watcher (MVP)

### Что возможно “сразу” одним EXE

Браузерный трекинг ActivityWatch делается через расширение `aw-watcher-web`. Инсталляторы Windows **не могут надежно поставить расширение автоматически** для обычного пользователя без enterprise-политик браузера/MDM.

Поэтому в MVP:
- агент ставится одним EXE
- локальный ActivityWatch продолжает работать
- uploader стартует сам
- инсталлятор (опционально) открывает страницу инструкций ActivityWatch по установке расширения

### Варианты автоустановки (enterprise)

Если у вас управляемые браузеры (Chrome/Edge/Firefox) — можно настроить автоустановку расширения через политики:
- Chrome/Edge: ExtensionInstallForcelist (через GPO/Intune)
- Firefox: policies.json (Enterprise Policies)

После этого расширение появится у пользователей автоматически, и `aw-watcher-web` начнет писать события в локальный `aw-server`.

### “Как ManicTime без расширения”

Это отдельная задача (не MVP). На Windows без расширения обычно нужны:
- прокси/VPN-агент и анализ URL на сетевом уровне, или
- сбор телеметрии через корпоративные механизмы/ETW/браузерные enterprise API,

и это заметно сложнее, чем интеграция через `aw-watcher-web`.
