# Telegram: отчёты и подписки

## Цель
Автоматически и по запросу отправлять отчёты по сотрудникам/командам в Telegram (группы/чаты).

## Типы отчётов (MVP → расширение)
- **Дневной отчёт (по всем пользователям организации)**\n
  - KPI% + индикатор (green/yellow/red/blue)\n
  - штраф за день (‑1000/‑3000)\n
  - отметка «Отсутствие» если KPI=blue или если день помечен типом отсутствия (Прогул/Отпуск/и т.д.)\n
- **Месячный отчёт (агрегированный)**\n
  - кол-во good/medium/bad/absence дней\n
  - рабочие/выходные/праздники\n

## Расписание
MVP:\n
- ежедневный job `tg_daily_report` (APScheduler) — **09:00 UTC**\n

Дальше (в ТЗ):\n
- настраиваемое время (по org) + отправка по таймзонам\n
- отдельные расписания по отделам/группам\n

## Формат сообщений (MVP)
HTML, одна простая «простыня», чтобы читалось в группе:\n
- Заголовок: `Отчёт за YYYY-MM-DD`\n
- Строки: `Имя: KPI% (цвет), штраф N`\n

Пример:\n
`Иван Петров: 54.5% (green), штраф 0`\n
`Мария: 22.1% (red), штраф -3000`\n

## Управление подписками (чаты/группы) в админке
Хранение:\n
- таблица `telegram_subscriptions` (org_id, chat_id, title, enabled)\n

API (уже есть):\n
- `GET /api/v1/telegram/subscriptions` — список\n
- `POST /api/v1/telegram/subscriptions` — добавить\n
- `POST /api/v1/telegram/send-daily-report?date=YYYY-MM-DD` — ручной запуск\n

UX в админке (в ТЗ):\n
- поле `chat_id` (для группы обычно отрицательное значение)\n
- кнопка «проверить отправку» (тестовое сообщение)\n
- включить/выключить подписку\n

## Безопасность и эксплуатация
- токен бота хранить только в env: `TELEGRAM_BOT_TOKEN`\n
- ограничить управление подписками ролью admin\n
